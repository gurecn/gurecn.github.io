正则判断字符串包含数组元素
正则表达式
Java
发布日期:   2022-02-10
作者:   Gure
文章字数:   1.5k

需求背景

项目要实现判断图片是否是截图。由于Android系统并没有提供获取方式，所以需要另辟蹊径实现。
经常使用安卓手机的用户可能发现一个规律，就是所有截图文件命名基本上都包含截图的几个特殊关键字，比如：screenshot、screenshots、screen_shot、screen-shot、screen shot、screencapture、screen_capture、screen-capture、screen capture、screencap、screen_cap、screen-cap、screen cap、截屏。此处所有字母都转换成了小写，有些手机会使用首字母大写或全大写。
那么问题来了，如何简洁、快速、有效的判断图片名中是否包含以上这些关键字呢？网上大部分使用了遍历，但是我看效率不高，除了遍历能不能使用正则表达式呢？
今天我们就使用正则表达式方式来实现判断。

实现过程

方式一使用遍历

首先我们使用遍历的方式来实现判断。
直接使用for循环判断即可。

public static void main(String[] args) {
    String[] words = {"screenshot", "screenshots", "screen_shot", "screen-shot", "screen shot", "screencapture", "screen_capture", "screen-capture", "screen capture"};
    
    String fimeName = "Screenshot_20220331_165242_com.huawei.android.launcher.jpg";
    for (String word : words) {
        if (fimeName.contains(word)) {
            System.out.println("for data:" + fimeName + " result:" + true);
            break;
        }
    }
}

执行发现结果如下：

for data:Screenshot_20220331_165242_com.huawei.android.launcher.jpg result:false

Process finished with exit code 0

判断是false，因为文件名是首字母大写。
怎么办呢？也好办，我们为他们统一转换一下格式即可。

public static void main(String[] args) {
    String[] words = {"screenshot", "screenshots", "screen_shot", "screen-shot", "screen shot", "screencapture", "screen_capture", "screen-capture", "screen capture"};
    String fimeName = "Screenshot_20220331_165242_com.huawei.android.launcher.jpg".toLowerCase();
    boolean result = false;
    for (String word:words){
        if(fimeName.contains(word.toLowerCase())){
            result = true;
            break;
        }
    }
    System.out.println("for data:" + fimeName + " result:" + result);
}

再次执行发现判断正常了：

for data:screenshot_20220331_165242_com.huawei.android.launcher.jpg result:true

Process finished with exit code 0

但是这种方式效率较差，且代码整洁度不够。
我们能否使用正则表达式判断呢？显然是可以的。

方式二使用正则匹配

我们可以使用Java提供的Pattern正则匹配的方式实现。
我们了解到：1.Pattern可以支持多个匹配规则；2.Matcher提供有三个返回判断的匹配方法：

matches：用于全字符串匹配，也就是目标字符串和匹配规则必须完全一样。
lookingAt：从目标字符串最开头开始匹配满足的子串，开头不满足直接失败，和startsWith类似。
find：对目标字符串的任意位置字符串匹配，也可以通过参数指定匹配起点。

通过以上两个特点，使用Pattern与Matcher完全能满足匹配了。

public static void main(String[] args) {
    String regex = "screenshot|screenshots|screen_shot|screen-shot|screen shot|screencapture|screen_capture|screen-capture|screen capture|screencap|screen_cap|screen-cap|screen cap|截屏";
    String fimeName = "Screenshot_20220331_165242_com.huawei.android.launcher.jpg";
    Pattern p = Pattern.compile(regex);

    boolean result = p.matcher(fimeName).find();
    System.out.println("find data:" + fimeName + " result:" + result);
}

此时执行结果如下：

find data:Screenshot_20220331_165242_com.huawei.android.launcher.jpg result:false

Process finished with exit code 0

发现竟然匹配失败了。原来问题依然出在大小写上，目标字符串首字母大写了。
此时有两种方式：1.将目标字符串转成小写匹配；2.使正则忽略大小写。
我们使用第二种方式：
compile方法可以设定匹配标记，flags可以配置成：

CASE_INSENSITIVE：忽略大小写。
MULTILINE：如果规则里出现^或者$，默认只会匹配第一行；设置了Multiline后会匹配所有行。我在Java测试没有配置好像也可以匹配所有行。
DOTALL：默认情况下DOT（.）不会匹配换行符。设置了Dotall模式会匹配所有字符，包括换行符。
UNICODE_CASE：如果指定则在CASE_INSENSITIVE启用时，将以符合Unicode标准的方式进行不区分大小写的匹配。默认情况下不区分大小写的匹配只匹配US-ASCII字符集中的字符。
CANON_EQ：启用规范等价。当指定此标志时，当且仅当两个字符的完整规范分解匹配时，才会认为两个字符匹配。例如当指定此标志时，表达式“a\u030A”将与字符串“\u00E5”匹配。默认情况下，匹配不考虑规范等价性。个人理解：匹配的字符有多种编码时，指定该标志，会转换成同一格式匹配；不指定不转换。所以指定时上面案例能匹配；否则不匹配。
UNIX_LINES：启用Unix行模式。
LITERAL：启用模式的文字解析。
UNICODE_CHARACTER_CLASS：启用Unicode版本的预定义字符类和POSIX字符类。POSIX字符类是许多正则表达式实现都支持的一种简写形式。
COMMENTS：允许模式中的空格和注释。

按照要求，修改代码如下：

public static void main(String[] args) {
    String regex = "screenshot|screenshots|screen_shot|screen-shot|screen shot|screencapture|screen_capture|screen-capture|screen capture|screencap|screen_cap|screen-cap|screen cap|截屏";
    String fimeName = "Screen_shot_20220331_165242_com.huawei.android.launcher.jpg";
    Pattern p = Pattern.compile(regex, Pattern.CASE_INSENSITIVE);

    boolean result = p.matcher(fimeName).find();
    System.out.println("find data:" + fimeName + " result:" + result);
}

此时运行，结果如下：

find data:Screen_shot_20220331_165242_com.huawei.android.launcher.jpg result:true

Process finished with exit code 0

以上代码就简单地实现了判断操作。

总结

最后我们再对比一下find和matches方法。

public static void main(String[] args) {
    String regex = "screenshot|screenshots|screen_shot|screen-shot|screen shot|screencapture|screen_capture|screen-capture|screen capture|screencap|screen_cap|screen-cap|screen cap|截屏";
    String fimeName = "Screen_shot";
    Pattern p = Pattern.compile(regex, Pattern.CASE_INSENSITIVE);

    boolean result = p.matcher(fimeName).find();
    System.out.println("find data:" + fimeName + " result:" + result);
    result = p.matcher(fimeName).matches();
    System.out.println("matches data:" + fimeName + " result:" + result);

    fimeName = "screen";
    result = p.matcher(fimeName).find();
    System.out.println("find fimeName:" + fimeName + " result:" + result);
    result = p.matcher(fimeName).matches();
    System.out.println("matches fimeName:" + fimeName + " result:" + result);

    fimeName = "screenshot123";
    result = p.matcher(fimeName).find();
    System.out.println("find fimeName:" + fimeName + " result:" + result);
    result = p.matcher(fimeName).matches();
    System.out.println("find matches:" + fimeName + " result:" + result);
    
    fimeName = "ScreenShot";
    result = p.matcher(fimeName).find();
    System.out.println("find fimeName:" + fimeName + " result:" + result);
    result = p.matcher(fimeName).matches();
    System.out.println("matches fimeName:" + fimeName + " result:" + result);
}

执行结果：

find data:Screen_shot result:true
matches data:Screen_shot result:true
find fimeName:screen result:false
matches fimeName:screen result:false
find fimeName:screenshot123 result:true
find matches:screenshot123 result:false
find fimeName:ScreenShot result:true
matches fimeName:ScreenShot result:true

Process finished with exit code 0
