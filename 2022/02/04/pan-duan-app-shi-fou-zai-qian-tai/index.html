判断APP是否在前台
应用
Android
发布日期:   2022-02-04
作者:   Gure
文章字数:   965
阅读时长:   4 分
阅读次数:  

背景说明

发现招商银行等应用内有一个比较好玩的功能：当在应用内截屏时，会主动产出一个窗口，询问用户是否需要咨询客服。这个功能很有趣，那么就在开源项目实现。
应用要实现在应用内截屏时，弹出提示，那么首先就需要实现判断当前是否处于前台。今天就实现不借助任何权限实现判断当前是否处在应用内（应用是否处于前台）。

实现思路

判断当前应用是否在前台，可以借助于多种方式。
第一种方式可以借助系统权限，获取当前应用顶层Activity，通过包名或进程来判断是否处于前台。
使用这种方式首先需要获取GET_TASKS权限。但是从Android L开始，谷歌开始对getRunningTasks和getRecentTasks等接口进行限制使用，默认情况下，普通的第三方应用（非系统应用）不能使用该接口，哪怕权限已经获取也会受到限制。
第二种方式可以借助应用自身维护Activity栈状态，通过自己维护的状态判断当前应用是否处于前台。
今天我们就实现不借助任何权限实现判断当前是否处在应用内（应用是否处于前台）。

实现过程

第一步定义接口

首先定义一个接口IApplication，定义管理Activity栈的抽象方法。

public interface IApplication {
  void addActivityToStack(Activity activity);

  void removeActivityFromStack(Activity activity);

  void addActivityStart(Activity activity);

  void addActivityPause(Activity activity);

  void finishAllActivity();

  boolean isForegroundStack();
}

六个抽象方法作用分别为：

addActivityToStack：Activity进栈方法，当应用中Activity调用onCreate时调用该方法。
removeActivityFromStack：Activity出栈方法，当应用中Activity调用onDestroy时调用该方法。
addActivityStart：Activity显示方法，当应用中Activity调用onStart时调用该方法。
addActivityPause：Activity隐藏方法，当应用中Activity调用onPause时调用该方法。
finishAllActivity：退出应用时，清空所有Activity。
isForegroundStack：判断当前应用是否在前台。

第二步创建FrontiaApplication

接下来我们创建FrontiaApplication，继承Application，同时实现IApplication接口。
在FrontiaApplication我们创建两个List列表activitiesInStack、activitiesInStart，分别保存栈队列Activity和当前处于Start状态的Activity。

public class FrontiaApplication extends Application implements IApplication {
  private static final String TAG = "FrontiaApplication";
  private List<Activity> activitiesInStack = new LinkedList<>();
  private List<Activity> activitiesInStart = new LinkedList<>();

  @Override
  public void addActivityToStack(Activity activity) {
    this.activitiesInStack.add(activity);
  }

  @Override
  public void removeActivityFromStack(Activity activity) {
    this.activitiesInStack.remove(activity);
  }

  @Override
  public void addActivityStart(Activity activity) {
    activitiesInStart.add(activity);
  }

  @Override
  public void addActivityPause(Activity activity) {
    activitiesInStart.remove(activity);
  }

  @Override
  public void finishAllActivity() {
    for (int i = 0; i < this.activitiesInStack.size(); i++) {
      Activity activity = this.activitiesInStack.get(i);
      if (activity != null && !activity.isFinishing()) {
        activity.finish();
      }
    }
    this.activitiesInStack.clear();
  }

  @Override
  public boolean isForegroundStack() {
    return activitiesInStart.size() > 0;
  }
}

第三步使用FrontiaApplication

使应用Application继承FrontiaApplication，或者直接将FrontiaApplication配置成应用Application。

public class TheApplication extends FrontiaApplication {
    private static TheApplication sInstance;

    public TheApplication() {
        sInstance = this;
    }

    public static TheApplication getApplication() {
        return sInstance;
    }
    //原Application内容。。。
}

第四步调用FrontiaApplication的方法

使应用内所有Activity在生命周期中onCreate、onStart、onPause、onDestroy调用FrontiaApplication中实现的方法。或者在所有Activity的父类BaseActivity对应生命周期中调用。

public class BaseActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        Application application = getApplication();
        if (application instanceof IApplication) {
            ((IApplication) application).addActivityToStack(this);
        }
    }

    @Override
    protected void onDestroy() {
        super.onDestroy();
        Application application = getApplication();
        if (application instanceof IApplication) {
            ((IApplication) application).removeActivityFromStack(this);
        }
    }

    @Override
    protected void onStart() {
        super.onStart();
        Application application = getApplication();
        if (application instanceof IApplication) {
            ((IApplication) application).addActivityStart(this);
        }
        MediaContentObserver.getInstance().stareObserve(this);
    }

    @Override
    protected void onPause() {
        super.onPause();
        Application application = getApplication();
        if (application instanceof IApplication) {
            ((IApplication) application).addActivityPause(this);
        }
    }
    
}

至此，所有代码都完成了。
当我们需要判断是否在前台时，直接判断方法即可：

boolean isForegroundStack = TheApplication.getApplication().isForegroundStack();
if(!isForegroundStack){
    //当前应用不在前台
    return;
}

总结

通过该方式，可以借助于FrontiaApplication进行扩展实现便捷的对应用管理。
同时可以实现在不需要申请任何权限的情况下，通过自己维护的队列状态判断当前应用是否处于前台。
