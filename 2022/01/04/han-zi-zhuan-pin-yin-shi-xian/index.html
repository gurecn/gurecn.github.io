需求背景

诗词模块需要实现拼音显示功能，类似与下面图片的效果：

那么就需要实现文字转拼音的功能。
实际在项目中，汉字转拼音的方案使用很普及，比如通讯录、消息等按照首字母排序，联系人显示拼音头像等。他们均需要实现将汉字转换成拼音内容。
今天我们就主要研究一下两种方式实现汉字转换成拼音。

实现方案

一、TinyPinyin方案：

TinyPinyin是一个适用于Java和Android的快速、低内存占用的汉字转拼音库。旨在优化使用更普及的pinyin4j库。
Github上对原理介绍的文档已经打不开了，我查询资料找到了，大家可以参考：

  • 打造最好的Java拼音库TinyPinyin（一）：单字符转拼音的极致优化
  • 打造最好的Java拼音库TinyPinyin（二）：多音字快速处理方案
  • 打造最好的Java拼音库TinyPinyin（三）：API设计和测试实践

大概原理是发现汉字规律：

  • 中文字符共有20378个
  • 中文字符除了12295外，均分布在 19968 ~ 40869 之间（Unicode的4E00 ~ 9FA5），并非连续分布，此范围内夹杂了524个非中文字符
  • 拼音共有407个（不包含声调）

接着构建映射表：

  • 使用byte[21000]存储每个汉字的低8位拼音编码，另外采用byte[21000/8]来存储每个汉字第9位（最高位）的编码，每个byte可存储8个汉字的第9位编码。

详细的原理请参考上面作者的博客。
TinyPinyin使用方式也很简单：
首先引入依赖：

dependencies {
    compile 'com.github.promeg:tinypinyin:2.0.3' // TinyPinyin核心包，约80KB
    compile 'com.github.promeg:tinypinyin-lexicons-android-cncity:2.0.3' // 可选，适用于Android的中国地区词典
    compile 'com.github.promeg:tinypinyin-lexicons-java-cncity:2.0.3' // 可选，适用于Java的中国地区词典
}

之后就可以调用转换接口：

/**
 * 如果c为汉字，则返回大写拼音；如果c不是汉字，则返回String.valueOf(c)
 */
String Pinyin.toPinyin(char c)

/**
 * c为汉字，则返回true，否则返回false
 */
boolean Pinyin.isChinese(char c)

/**
 * 将输入字符串转为拼音，转换过程中会使用之前设置的用户词典，以字符为单位插入分隔符
 */
String toPinyin(String str, String separator)

经作者对比有以下优势：

  • 支持基于词典的多音字处理，支持简体中文、繁体中文；
  • 极速的执行效率（Pinyin4J的4~16倍）；
  • 很低的内存占用（不添加词典时小于30KB）。

但是TinyPinyin也有很明显的劣势：

  • 不支持繁体中文；
  • 不支持声调注音。

因为诗词想要添加声调显示，所以我最终使用了第二种方案pinyin4j。

二、pinyin4j方案：

Pinyin4j是一个流行的Java库，支持中文字符和拼音之间的转换，且拼音输出格式可以定制。
与TinyPinyin对比，Pinyin4J的优势：

  • 支持繁体中文字符；
  • 支持转换到汉语拼音，通用拼音, 威妥玛拼音（威玛拼法）, 注音符号第二式, 耶鲁拼法和国语罗马字；
  • 支持多音字，即可以获取一个中文字符的多种发音；
  • 支持多种字符串输出格式，比如支持Unicode格式的字符ü和声调符号（阴平 “ˉ”,阳平”ˊ”,上声”ˇ”,去声”ˋ”）的输出。

使用Pinyin4J方式流程如下：
首先添加依赖：

implementation files('libs\\pinyin4j-2.5.0.jar')

然后定义输出格式：

HanyuPinyinOutputFormat format = new HanyuPinyinOutputFormat();
format.setCaseType(HanyuPinyinCaseType.LOWERCASE);
format.setToneType(HanyuPinyinToneType.WITH_TONE_MARK);
format.setVCharType(HanyuPinyinVCharType.WITH_U_UNICODE);

最后进行数据转换：

StringBuilder sb = new StringBuilder();
try {
  for(char cha:hanzi.toCharArray()){
    if(Character.toString(cha).matches("[\\u4E00-\\u9FA5]+")) {
      String[] pys = PinyinHelper.toHanyuPinyinStringArray(cha, format);
      sb.append(pys[0]).append(" ");
    }
  }
  sb.deleteCharAt(sb.length()-1);
} catch (Exception exception){
  exception.printStackTrace();
}
return sb.toString();

其中HanyuPinyinOutputFormat中调用的三个配置方法注释如下：

setCaseType()
输出汉语拼音的大小写
HanyuPinyinCaseType.UPPERCASE：设置为大写
HanyuPinyinCaseType.LOWERCASE：设置为小写
setVCharType()
设置字符V（ü）的显示方式
HanyuPinyinVCharType.WITH_U_AND_COLON：显示为u:（默认）
HanyuPinyinVCharType.WITH_V ：显示为v
HanyuPinyinVCharType.WITH_U_UNICODE：显示为ü
setToneType()
设置汉语拼音的声调
HanyuPinyinToneType.WITH_TONE_NUMBER：使用数字标识声调，例如有（yŏu）显示为（you3）。（默认）
HanyuPinyinToneType.WITHOUT_TONE：设置不显示声调，例如有（yŏu）显示为（you）
HanyuPinyinToneType.WITH_TONE_MARK：设置显示声调，例如有（yŏu）显示为（yŏu）。此时setVCharType必须设置WITH_U_UNICODE，否则会抛出异常。
