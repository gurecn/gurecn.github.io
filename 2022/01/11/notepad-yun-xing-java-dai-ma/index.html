Notepad++运行JAVA代码
工具
Java
发布日期:   2022-01-11
作者:   Gure
文章字数:   703
阅读时长:   2 分
阅读次数:  

有时候我们会临时写一些简单的Java代码执行，验证执行结果。不一定适合使用IDEA这样比较繁重的工具，但是直接使用记事本编写，采用命令执行又比较复杂。这时候类似于Notepad++、EditPlus这类常用的文本编辑工具就可以临时顶上来了。
现在Notepad++、EditPlus等都支持直接运行Java、C等代码（开发环境需单独安装）。
因为我现在用的比较多的是Notepad++（免费），所以我们就以Notepad++为例，简单配置一下如何再Notepad++运行Java代码。
我使用的是Windows 11，其他系统操作类似。
准备工作：

  • 安装Notepad++，参考官网。
  • 安装Java环境，参考：Java 开发环境配置。

开始进行相关配置：
首先打开Notepad++，在工具栏依次打开：插件-插件管理，勾选NPPExec，点击右上角安装。
若发现界面语言是英语，可以先把界面语言切换成中文，在工具栏依次点击：Settings-Preferences-General-Localization，选择中文简体。然后再进行插件下载。
插件下载后，在工具栏依次点击插件-NPPExec-Excute NppExec Script…，在输入框输入：

NPP_SAVE
javac.exe -encoding UTF-8 "$(FILE_NAME)"
java.exe "$(NAME_PART)"

此时点击Save…，弹出框中重新命名：Java编译加执行。再点击Save进行保存，最后点击OK。此时若打开的文件不是可执行的Java程序，界面会弹出控制台，输出：

NPP_SAVE: 新文件 1
javac.exe -encoding UTF-8 "新文件 1"
Process started (PID=12040) >>>
javac: 无效的标记: 新文件 1
用法: javac <options> <source files>
-help 用于列出可能的选项
<<< Process finished (PID=12040). (Exit code 2)
java.exe "新文件 1"
Process started (PID=10196) >>>
错误: 找不到或无法加载主类 新文件 1
<<< Process finished (PID=10196). (Exit code 1)
================ READY ================

此时打开的程序若是Java程序，则会执行一次，并在控制台输出结果：
我写的测试程序如下：

public class HelloWorld{
    public static void main(String[] args){
        System.out.print("Hello World!你好啊");
    }
}

输出结结果如下：

NPP_SAVE: C:\Users\Administrator\Desktop\HelloWorld.java
javac.exe -encoding UTF-8 "HelloWorld.java"
Process started (PID=2604) >>>
<<< Process finished (PID=2604). (Exit code 0)
java.exe "HelloWorld"
Process started (PID=9676) >>>
Hello World!你好啊<<< Process finished (PID=9676). (Exit code 0)
================ READY ================

此时配置就完成了。
后续执行java程序有两种方式：

  • 可以使用快捷键F6，在弹出窗口中点击OK执行。
  • 可以在工具栏，点击Excute NppExec Script按钮，在弹出窗口中点击OK执行。

我们也可以在工具栏依次点击插件-NPPExec-Excute Advanced Options中进行包括菜单在内的各项设置。
