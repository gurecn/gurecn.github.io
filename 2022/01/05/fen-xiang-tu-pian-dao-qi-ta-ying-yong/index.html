分享图片到其他应用
三方资源
Android
发布日期:   2022-01-05
作者:   Gure
文章字数:   1.5k
阅读时长:   6 分
阅读次数:  

前几篇文章实现了将诗词显示的效果保存到图片中，并显示到图库里。
今天我们实现一下将图片分享到其他应用中。
实现分享功能的方式有几种，今天我们主要使用两种方式实现：

方法一

我们首先使用系统提供的createChooser方式方式实现。
Android系统ACTIVITY_INTENT_ACTION中定义有ACTION_SEND，我们查看描述：

Activity Action: Deliver some data to someone else. Who the data is being delivered to is not specified; it is up to the receiver of this action to ask the user where the data should be sent.
When launching a SEND intent, you should usually wrap it in a chooser（through createChooser）, which will give the proper interface for the user to pick how to send your data and allow you to specify a prompt indicating what they are doing.

翻译：

Activity操作：将一些数据传送给其他人。未指定数据接收者，由这个动作的接收者来询问用户应该将数据发送到哪里。
当启动一个 SEND 意图时，你通常应该将它包装在一个选择器中（通过 createChooser），这将为用户提供适当的界面来选择如何发送你的数据，并允许你指定一个提示，指示他们正在做什么。

根据描述，我们可以借助于ACTION_SEND实现将图片转发给其他应用。
分享图片实现方式如下：

/**
 * 分享应用
 */
public void shareImage(Uri imageUri) {
  Intent shareIntent = new Intent();
  String shareAppText = getResources().getString(R.string.share_app_text);
  shareIntent.setAction(Intent.ACTION_SEND);
  shareIntent.putExtra(Intent.EXTRA_TITLE,shareAppText);
  shareIntent.putExtra(Intent.EXTRA_TEXT, shareAppText);
  shareIntent.putExtra(Intent.EXTRA_STREAM, imageUri);
  shareIntent.setType("image/");
  startActivity(Intent.createChooser(shareIntent, "分享到"));
}

ACTION_SEND同时支持多宗类型数据，比如：text、video、audio、application等等。
使用该方式优势如下：

  • 无需依赖其他资源，直接系统调用；
  • 开发方式简单，系统兼容性强；

劣势如下：

  • 无法定制显示效果；
  • 无法指定应用（可设置过滤器，但需申请应用管理权限）；
  • 分享的数据效果可能不同。发现微信显示的图片效果与集成微信分享不同。

方法二

采用集成各个应用单独提供的SDK实现分享。
该方案比较麻烦，支持一款应用需要单独到对应应用开放平台注册账号、创建应用、下载对应SDK。我们以微信为例：
第一步，注册并登录微信开放平台（需注意，不是微信公众平台、腾讯开放平台、QQ开放平台等，厂大就是好，光开放平台就好几个）。
第二步，创建移动应用。就想要实现一个微信分享，需要上传n多信息，搞得我以为直接把我的应用给我上架发布一样，强烈吐槽。
第三步，获取微信开放平台生成的AppID，并下载集成SDK和Demo，按照Demo进行集成。
细节不讲了，demo很详细了，我这里就贴一下我使用的代码：

/**
 * 分享微信
 * @param scene 模式：聊天、朋友圈
 */
private void shareWechat(int scene) {
  IWXAPI api = WXAPIFactory.createWXAPI(this, "平台生成的AppID值",false);
  WXImageObject imgObj = new WXImageObject(ShiciActivity.mBitmap);
  WXMediaMessage msg = new WXMediaMessage();
  msg.mediaObject = imgObj;
  SendMessageToWX.Req req = new SendMessageToWX.Req();
  req.transaction = "shareimg";
  req.message = msg;
  req.scene = scene;
  boolean isSend =  api.sendReq(req);
  Log.d(TAG, "isSend:" + isSend);
}

做完以上步骤后，就可以实现图片分享到微信及朋友圈中了。钉钉分享实现方式类似。
在创建移动应用时，有一点需要注意，就是需要填写应用签名。
应用签名是打包应用时使用的jks文件，此处需要填写MD5值。我遇到一个问题：
使用keytool -list -v -keystore命令获取签名信息，竟然没有MD5:

可以看到里面只包含SHA1、SHA256，我把它们分别填入微信开放平台发现不生效，分享时提示签名校验失败。
为什么呢？
查阅资料发现，很多人遇到这个问题，普遍反馈是jdk的问题。解决方式有以下几个：

  • 降低jdk版本。1.8.0及以前的版本使用该命令可以获取正确的MD5。但是该方案需要安装旧版jdk，我不是很建议。
  • 使用微信提供的签名获取工具获取。
  • 使用反编译工具反编译apk，能够获取签名值。

由于我电脑安装了反编译工具jadx，我直接使用第三种方式获取签名如下：

其中MD5 签名就是所需的签名值。需要注意的是，填写时需要去掉中间的空格。使用keytool 获取时，需要去除中间的“:”。

根据项目实际需要，我最终选择了第二种方案，实现了图片对微信、朋友圈、钉钉分享能力。
