视图绑定viewBinding
特性 Android
Java
发布日期:   2022-03-02
作者:   Gure
文章字数:   1.5k
阅读时长:   5 分
阅读次数:  

背景

今天调试项目时，创建一个新工程，发现Activity中并没有看到熟悉的setContentView(R.layout.activity_xx)代码。更奇怪的是对应的Activity还能加载上相应的layout布局文件。当时差点惊掉我的眼睛。
怎么回事呢？
原来这就是 Android Studio 3.6 Canary 11 及更高版本中新增的特新-视图绑定。

研究

开启视图绑定

开启视图绑定功能，需要有以下前提条件：
使用Android Studio 3.6 Canary 11 及更高版本；
同时Android 插件需要使用以下工具：

  • Gradle 5.6.4。 如需了解详情，请阅读有关更新 Gradle 的部分。
  • SDK Build Tools 28.0.3 或更高版本。

配置视图绑定

视图绑定功能可按模块启用。要在某个模块中启用视图绑定，需要将 viewBinding 元素添加到对应模块的build.gradle 文件中。

android {
    compileSdk 31
    
    defaultConfig {
        applicationId "com.gurecn.dragfloatbutton"
        minSdk 21
        targetSdk 31
        versionCode 1
        versionName "1.0"
    }
    
    buildTypes {
        release {
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
    
    buildFeatures {
        viewBinding true
    }
}

此时重新同步Gradle，编译成功配置生效后，系统会为该模块中包含Layout路径下的每个 XML 布局文件生成一个绑定类（比如activity_main.xml则生成一个ActivityMainBinding）。系统会通过以下方式生成绑定类的名称：将 XML 文件的名称转换为驼峰式大小写，并在末尾添加“Binding”一词。
每个绑定类均包含对根视图以及具有 ID 的所有控件的引用。

创建布局文件

我们在layout路径下创建布局文件layout_test.xml，布局文件中添加几个控件。

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_height="match_parent"
    android:layout_width="match_parent"
    xmlns:app="http://schemas.android.com/apk/res-auto">
    <com.gurecn.dragfloatbutton.LableSettingsView
        android:id="@+id/lsv_test"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:layout_alignParentBottom="true"
        app:itemLableClickable="true"
        app:itemLableName="LableSettingsView"
        app:itemLableSummary="这是一条自定义View"/>

</LinearLayout>

使用视图绑定

此时我们可以在Activity中使用layout_test.xml文件对应的视图绑定。根据命名规则，视图绑定的名称为：LayoutTestBinding。
此时我们使用该视图绑定进行试图初始化：

public class MainActivity extends AppCompatActivity {
    private LayoutTestBinding binding;
    
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        binding = LayoutTestBinding.inflate(getLayoutInflater());
        setContentView(binding.getRoot());
    }
}

对应开篇说的setContentView(R.layout.activity_xx)代码，此时我们使用setContentView\(binding.getRoot())替代。getRoot() 方法会返回布局文件的根视图。
对于Fragment 使用时，我们会使用另一个初始化视图的方式：

public class FirstFragment extends Fragment {
  private LayoutTestBinding binding;

  @Override
  public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
    binding = LayoutTestBinding.inflate(inflater, container, false);
    return binding.getRoot();
  }
  
  @Override
  public void onDestroyView() {
    super.onDestroyView();
    binding = null;
  }
}

注意：Fragment 的存在时间比其视图长。请务必在 Fragment 的 onDestroyView() 方法中清除对绑定类实例的所有引用。

使用控件

当我们使用布局中的控件时，我们可以直接通过视图绑定LayoutTestBinding中的控件引用来直接操作。由于layout_test.xml布局文件只有一个添加ID的控件，所以LayoutTestBinding具有一个字段：一个是名为 lsvTest的 LableSettingsView控件。
需要注意的是：当该布局中有其他控件没有添加 ID时，这些控件是不会在LayoutTestBinding中创建引用的。

public class MainActivity extends AppCompatActivity {
  private LayoutTestBinding binding;

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    binding = LayoutTestBinding.inflate(getLayoutInflater());
    setContentView(binding.getRoot());
    binding.lsvTest.setOnClickListener(new View.OnClickListener() {
      @Override
      public void onClick(View v) {
        Snackbar.make(v, "自定义控件", Snackbar.LENGTH_LONG).show();
      }
    });
  }
}

忽略绑定视图

若是有些布局文件，我们无需为其生成绑定视图。那么我们可以使用viewBindingIgnore主动声明为其不生成绑定视图。

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:layout_height="match_parent"
    android:layout_width="match_parent"
    tools:viewBindingIgnore="true">
    <com.gurecn.dragfloatbutton.LableSettingsView
        android:id="@+id/lsv_test"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:layout_alignParentBottom="true"
        app:itemLableClickable="true"
        app:itemLableName="LableSettingsView"
        app:itemLableSummary="这是一条自定义View"/>
</LinearLayout>

总结

视图绑定的特点有哪些呢？

与findViewById对比

与使用 findViewById 相比，视图绑定具有一些很显著的优点：

  • Null 安全：由于视图绑定会创建对视图的直接引用，因此不存在因视图 ID 无效而引发 Null 指针异常的风险。此外，如果视图仅出现在布局的某些配置中，则绑定类中包含其引用的字段会使用 @Nullable 标记。
  • 类型安全：每个绑定类中的字段均具有与它们在 XML 文件中引用的视图相匹配的类型。这意味着不存在发生类转换异常的风险。

这些差异意味着布局和代码之间的不兼容将会导致构建在编译时（而非运行时）失败。

与数据绑定对比

视图绑定和数据绑定均会生成可用于直接引用视图的绑定类。但是，视图绑定旨在处理更简单的用例，与数据绑定相比，具有以下优势：

  • 更快的编译速度：视图绑定不需要处理注释，因此编译时间更短。
  • 易于使用：视图绑定不需要特别标记的 XML 布局文件，因此在应用中采用速度更快。在模块中启用视图绑定后，它会自动应用于该模块的所有布局。

反过来，与数据绑定相比，视图绑定也具有以下限制：

视图绑定不支持布局变量或布局表达式，因此不能用于直接在 XML 布局文件中声明动态界面内容。
视图绑定不支持双向数据绑定。

考虑到这些因素，在某些情况下，最好在项目中同时使用视图绑定和数据绑定。您可以在需要高级功能的布局中使用数据绑定，而在不需要高级功能的布局中使用视图绑定。
关于数据绑定的研究，我们找时间再进行。
