HarmonyOS应用WebView网页交互
应用
HarmonyOS
发布日期:   2022-08-15
作者:   Gure
文章字数:   1.5k
阅读时长:   6 分
阅读次数:  

研究HarmonyOS应用开发，首先考虑做一个集成个人博客的客户端。
可以直接使用WebView，功能很简单，所以编码也很快。
WebView的介绍和使用可以参考官方开发文档。
下面我们介绍一下WebView的使用和遇到的问题。

WebView实现

申请网络权限

WebView显示网页内容，首先需要申请网络前线，确保能够联网获取数据。
我们依次打开：entry > src > main > config.json,并在module项下添加如下配置。

"module": {
  "package": "com.devdroid.web",
  "name": ".MyApplication",
  "reqPermissions": [
    {
      "name": "ohos.permission.INTERNET"
    }
  ],
  "distro": {
    "deliveryWithInstall": true,
    "moduleName": "entry",
    "moduleType": "entry",
    "installationFree": false
  },
  ...
}

创建布局文件

在layout目录下的xml文件中创建WebView。
我们找到MainAbilitySlice中setUIContent设置的布局文件。可以直接Ctrl + 左键跳转，也可以依次到resources > base > layout中找到对应的布局文件。
在布局文件中定义WebView控件，并且配置id。

<?xml version="1.0" encoding="utf-8"?>
<DirectionalLayout
  xmlns:ohos="http://schemas.huawei.com/res/ohos"
  ohos:height="match_parent"
  ohos:width="match_parent"
  ohos:alignment="center"
  ohos:orientation="vertical">
  <ohos.agp.components.webengine.WebView
    ohos:id="$+id:web_view"
    ohos:height="match_parent"
    ohos:width="match_parent"
    ohos:visibility="visible"/>
</DirectionalLayout>

加载Web页面

我们返回MainAbilitySlice中，在onActive方法中加载Web页面内容。

@Override
public void onActive() {
    super.onActive();
    ohos.agp.components.webengine.WebView web = (ohos.agp.components.webengine.WebView) findComponentById(ResourceTable.Id_web_view);
    web.load("https://gurecn.gitee.io/");
}

此时主要开发工作完成了，我们安装并启动应用查看情况。

问题

问题一 页面不响应点击事件

发现加载出来的Web页面效果显示不全，并且无法响应点击跳转事件。
按照Android开发的经验，web View需要setJavaScriptEnabled才能激活JavaScript事件响应。HarmonyOS同样需要启用JavaScript功能。
我们添加配置如下。

@Override
public void onActive() {
    super.onActive();
    ohos.agp.components.webengine.WebView web = (ohos.agp.components.webengine.WebView) findComponentById(ResourceTable.Id_web_view);
    web.getWebConfig().setJavaScriptPermit(true);
    web.load("https://gurecn.gitee.io/");
}

重新启动后，发现一部分控件点击能够响应，提示点击效果并跳转。
但是Web页面存在显示不全的情况。

问题二 页面显示不全

此时发现部分图片、条目显示空白，但是点击空白处能够正常进行跳转。说明JavaScript已经生效，但是部分数据没有显示，不显示的内容主要是图片等文件。
但是我们将网页地址换成百度https://www.baidu.com/时显示效果和点击都可以。难道是因为HarmonyOS对Hexo主题兼容性存在问题？
研究Android中WebView的使用，发现还需要配置开启缓存功能setDomStorageEnabled。HarmonyOS中同样有类似的需求。我们需要配置WebView支持访问Data Ability资源的能力。

@Override
public void onActive() {
    super.onActive();
    ohos.agp.components.webengine.WebView web = (ohos.agp.components.webengine.WebView) findComponentById(ResourceTable.Id_web_view);
    web.getWebConfig().setJavaScriptPermit(true);
    web.getWebConfig().setWebStoragePermit(true);
    web.load("https://gurecn.gitee.io/");
}

再次启动应用，会发现显示和点击功能都已经正常了。

问题三 点击跳转浏览器

但是使用中会发现新的问题，就是点击超链接时并不是在当前应用打开，而是会打开浏览器打开。
我想实现在APP内部跳转新的网页，而不是打开浏览器跳转。
此时可以借助于WebAgent实现。
我们只需要给WebView配置一个WebAgent即可。WebAgent对象内有很多可重写方法，以实现特定的目的。此时仅实现应用内跳转网页，而不是使用浏览器打开，我们不需要重写任何方法。

@Override
public void onActive() {
    super.onActive();
    ohos.agp.components.webengine.WebView web = (ohos.agp.components.webengine.WebView) findComponentById(ResourceTable.Id_web_view);
    web.getWebConfig().setJavaScriptPermit(true);
    web.getWebConfig().setWebStoragePermit(true);
    web.setWebAgent(new WebAgent(){
    });
    web.load("https://gurecn.gitee.io/");
}

问题四 返回时直接退出应用

此时发现当点击超链接时，会在当前APP内部直接跳转新的页面，不会启动浏览器打开。
但是同时又发现新的问题，就是执行返回（手势右滑、点击手机返回键）操作时，APP会直接退出。
分析原因也很明了，因为我们页面跳转都是在同一个Ability,页面切换时仅将WebView重新加载新Web页面内容，并没有启动新的Ability。返回时Ability监听到动作出栈，从而应用退出了。
此时我们可以重写onKeyDown方法，当判断用户执行返回操作时，判断WebView是否可以向后或向前浏览，若可以时使用goBack()或goForward()实现WebView向后或向前浏览。

@Override
public void onActive() {
    super.onActive();
    ohos.agp.components.webengine.WebView web = (ohos.agp.components.webengine.WebView) findComponentById(ResourceTable.Id_web_view);
    web.getWebConfig().setJavaScriptPermit(true);
    web.getWebConfig().setWebStoragePermit(true);
    web.setWebAgent(new WebAgent(){
    });
    web.load("https://gurecn.gitee.io/");
    webNavigator = web.getNavigator();
}

@Override
public boolean onKeyDown(int keyCode, KeyEvent keyEvent) {
    if (keyCode == KeyEvent.KEY_BACK && webNavigator.canGoBack()) {
        webNavigator.goBack();//返回上个页面
        return true;
    } else if (keyCode == KeyEvent.KEY_FORWARD && webNavigator.canGoForward()) {
        webNavigator.goForward();//返回下个页面
        return true;
    }
    return super.onKeyDown(keyCode, keyEvent);//退出整个Ability
}

问题五 Web自动调用APP打开

当浏览普通博客时没有异常，但是一旦博客跳转到一些第三方网站（简书、知乎）等时，会发现WebView会自动跳转到一些非http或https的url地址，类似：zhihu://answer/2580420308?utm_source=mobile_web_scheme。此时WebView就会提示：网页无法打开....。
这是因为此时知乎、简书等平台调用知乎客户端、简书客户端等软件打开对应的文章，因为WebView无法解析对应的协议，所以会提示错误。
此时我们可以使用问题三介绍的WebAgent实现对非http或https的url地址进行拦截，使WebView仅加载http或https的url地址。

@Override
public void onActive() {
    super.onActive();
    ohos.agp.components.webengine.WebView web = (ohos.agp.components.webengine.WebView) findComponentById(ResourceTable.Id_web_view);
    web.getWebConfig().setJavaScriptPermit(true);
    web.getWebConfig().setWebStoragePermit(true);
    web.setWebAgent(new WebAgent(){
        @Override
        public boolean isNeedLoadUrl(WebView webView, ResourceRequest request) {
            String url = request.getRequestUrl().toString();
            if(!url.startsWith("http:") && !url.startsWith("https:")){
                return false;
            }
            return super.isNeedLoadUrl(webView, request);
        }
    });
    web.load("https://gurecn.gitee.io/");
    webNavigator = web.getNavigator();
}

@Override
public boolean onKeyDown(int keyCode, KeyEvent keyEvent) {
    if (keyCode == KeyEvent.KEY_BACK && webNavigator.canGoBack()) {
        webNavigator.goBack();//返回上个页面
        return true;
    } else if (keyCode == KeyEvent.KEY_FORWARD && webNavigator.canGoForward()) {
        webNavigator.goForward();//返回下个页面
        return true;
    }
    return super.onKeyDown(keyCode, keyEvent);//退出整个Ability
}

此时再跳转对应的平台时，页面和内容已经能够正常显示了。
