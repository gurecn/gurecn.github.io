生成目录树形结构(Code Structure)
日常
技能
发布日期:   2021-11-11
作者:   Gure
文章字数:   1.2k
阅读时长:   5 分
阅读次数:  

一次小教训，让自己深刻明白了记录的重要性。

原因

由于开源项目中要非常直观展现项目的结构，为不同模块添加注释，便于其他人理解。所以我想要导出开源项目代码的整体结构。

过程

因为之前整理公司产品技术文档时已经导出过相关代码结构，所以这一次自然而然想到使用同样的方案。

但因为时间较久远，一时竟然想不起当时如何导出了（摊手，好脑袋不如烂笔头）。

由于是Android Studio项目，所以我第一时间打开IDEA在工具中查找方式，很明显没有找到对应的方案。然后就经历了漫长的探索之路：百度、谷歌、Android官方、jetbrains官方查询相关方式。

当然最终均已失败告终。

中间方案

当然中间也不是完全没有收获的，在网络中发现一个命令：Tree，借助该命令可以生成一个文件夹及子文件的路径（我电脑是Windows系统）。

经过尝试，首先切到项目路径下，然后执行对应命令：

tree /f

发现确实输出了目录结构。

C:.
└─com
    └─devdroid
        └─sleepassistant
            ├─activity
            │      AboutActivity.java
            │      AgreementAvtivity.java
            │      AppLockTimeActivity.java
            │      ChartActivity.java
            │      FeedbackActivity.java
            │      GuideActivity.java
            │      MainActivity.java
            │      RestrictionActivity.java
            │      SettingsActivity.java
            │
            ├─adapter

但是生成的项目结构和我希望的及以前生成的都不太一样。区别就是最后一级（显示文件时）文件名没有“|”及“-”。这让强迫症的我看的是很不舒服，但不能一个个添加吧。

最终方式

今天忽然灵机一动，有没有可能是Windows就支持这一种方式，但是Linux或者其他工具下同样的命令就输出不一样结果呢？

我电脑安装有Git Bash，里面安装有MINGW64，就自己验证一下。

首先使用npm命令安装tree-cli:

npm install -g tree-cli

$ npm install -g tree-cli
C:\Users\Administrator\AppData\Roaming\npm\tree -> C:\Users\Administrator\AppData\Roaming\npm\node_modules\tree-cli\bin\tree
C:\Users\Administrator\AppData\Roaming\npm\treee -> C:\Users\Administrator\AppData\Roaming\npm\node_modules\tree-cli\bin\tree
+ tree-cli@0.6.7
added 72 packages from 57 contributors in 3.658s

然后使用Tree命令，输出结构：

tree -l 5

C:\develop\DevProject\Android\Sleepassistant\app\src\main\java
└── com
   └── devdroid
      └── sleepassistant
         ├── activity
         |  ├── AboutActivity.java
         |  ├── AgreementAvtivity.java
         |  ├── AppLockTimeActivity.java
         |  ├── ChartActivity.java
         |  ├── FeedbackActivity.java
         |  ├── GuideActivity.java
         |  ├── MainActivity.java
         |  ├── RestrictionActivity.java
         |  └── SettingsActivity.java
         ├── adapter

总结

  1. 遇到问题找到解决方案时，一定要及时做笔记记录。不然过几天你就要再次经历辛苦的探究过程了。
  2. 咱们看一下tree-cli这个工具。
  • 根据官方说明：

Tree is a recursive directory listing program that produces a depth indented listing of files.With no arguments, tree lists the files in the current directory. When directory arguments are given, tree lists all the files and/or directories found in the given directories each in turn. Upon completion of listing all files/directories found,tree returns the total number of files and/or directories listed.

翻译一下就是：

Tree是一个递归目录列表程序，它生成文件的深度缩进列表。当没有参数时，Tree列出当前目录中的文件。当给定目录参数时，树依次列出在给定目录中找到的所有文件和/或目录。完成列出找到的所有文件/目录后，tree返回列出的文件和/或目录总数。

  • tree-cli可以指定的参数：

–help, -h

​ outputs a verbose usage listing.输出详细的用法列表。

–version

​ outputs the version of tree-cli.输出tree-cli的版本。

–debug

​ show debug info.显示调试信息。

–ignore

​ ignores directory or file you specify.忽略您指定的目录或文件

–base

​ specify a root directory. Relative path from cwd root and absolute path are both acceptable.

​ 指定根目录。来自cwd根的相对路径和绝对路径都是可以接受的。

–fullpath

​ prints the full path prefix for each file.打印每个文件的完整路径前缀。

–noreport

​ omits printing of the file and directory report at the end of the tree listing and omits printing the tree on console.省略在树列表末尾打印文件和目录报告，并省略在控制台上打印树。

-a

​ all files are printed. By default tree does not print hidden files (those beginning with a dot ‘.’). In no event does tree print the file system constructs ‘.’ (current directory) and ‘..’ (previous directory).打印所有文件。默认情况下，tree不显示隐藏文件（以点“.”开头的文件）。在任何情况下，树都不会打印文件系统构造 ‘.’ （当前目录）和“..”（上一目录）。

-d

​ list directories only.仅列出目录。

–directoryFirst

​ list directories first.优先列出目录。

-f

​ append a ‘/‘ for directories, a ‘=’ for socket files and a ‘|’ for FIFOs.为目录追加“/”，为socket文件追加“=”和为FIFO文件追加“|”。

-i

​ makes tree not print the indentation lines, useful when used in conjunction with the -f option.使树不打印缩进线，与 -f 选项结合使用时很有用。

-l

​ max display depth of the directory tree.目录树的最大显示深度。

-o

​ send output to filename.将输出发送到文件名。

终于，期待已久的结果出现了，赶紧记下来，别再忘记了。
